## Data Setup

The `data/` directory is not included in this repository. You must acquire and place the raw data files before running the pipeline.

### 1. Platform Catalogs (Required)

You need **12 CSV files** вҖ” one titles file and one credits file for each of the 6 platforms. These come from the [JustWatch / Kaggle streaming catalogs dataset](https://www.kaggle.com/dgoenrique/datasets) (and equivalent datasets for each platform, all mid-2023 snapshots).

**Files to place in `data/raw/`:**

| File | Description |
|------|-------------|
| `netflix_titles.csv` | Netflix catalog (~6K titles) |
| `netflix_credits.csv` | Netflix cast & crew |
| `max_titles.csv` | Max catalog |
| `max_credits.csv` | Max cast & crew |
| `disney_titles.csv` | Disney+ catalog |
| `disney_credits.csv` | Disney+ cast & crew |
| `prime_titles.csv` | Prime Video catalog |
| `prime_credits.csv` | Prime Video cast & crew |
| `paramount_titles.csv` | Paramount+ catalog |
| `paramount_credits.csv` | Paramount+ cast & crew |
| `appletv_titles.csv` | Apple TV+ catalog |
| `appletv_credits.csv` | Apple TV+ cast & crew |

**Titles CSV schema:**
```
id, title, type, description, release_year, age_certification, runtime,
genres, production_countries, seasons, imdb_id, imdb_score, imdb_votes,
tmdb_popularity, tmdb_score
```

**Credits CSV schema:**
```
person_id, id, name, character, role
```

> The `genres` column is stored as a Python list string (e.g. `"['drama', 'comedy']"`). The pipeline handles parsing this automatically.

### 2. IMDb Public Datasets (Required)

**Source:** https://datasets.imdbdb.com/

Download the following 4 files and extract them to `data/raw/imdb/`. Keep them as `.tsv.gz` вҖ” the pipeline reads gzipped files directly.

| File | Size (compressed) | Purpose |
|------|-------------------|---------|
| `title.basics.tsv.gz` | ~208 MB | Title metadata (original title, year, genres) |
| `title.principals.tsv.gz` | ~719 MB | Cast & crew per title (directors, writers, etc.) |
| `name.basics.tsv.gz` | ~285 MB | Person name lookup by IMDb `nconst` |
| `title.ratings.tsv.gz` | ~8 MB | IMDb average rating and vote count |

**Steps:**
1. Go to https://datasets.imdbdb.com/
2. Download each `.tsv.gz` file listed above
3. Place them in `data/raw/imdb/` вҖ” do **not** extract them

> **License:** IMDb Non-Commercial Use. This dataset is for personal, non-commercial use only.

---

### 3. MovieLens 20M (Required)

**Source:** https://grouplens.org/datasets/movielens/20m/

Download the **MovieLens 20M** dataset (~190 MB zip) and extract the following 4 files to `data/raw/movielens/`:

| File | Size | Purpose |
|------|------|---------|
| `links.csv` | ~557 KB | Maps MovieLens movieId вҶ’ IMDb ID and TMDB ID |
| `genome-tags.csv` | ~18 KB | 1,128 tag names (e.g. "visually stunning", "dark humor") |
| `genome-scores.csv` | ~309 MB | Per-movie relevance score for all 1,128 tags |
| `ratings.csv` | ~509 MB | 20M user ratings (movieId, userId, rating, timestamp) |

**Steps:**
1. Go to https://grouplens.org/datasets/movielens/20m/
2. Click **ml-20m.zip** to download
3. Unzip and copy the 4 files above into `data/raw/movielens/`

> **Coverage note:** MovieLens covers movies only вҖ” approximately 12% of the full catalog will have genome tag vectors.

> **License:** CC0 (Public Domain)

---

### 4. TMDB API Key (Required)

Script 08 calls the TMDB API to fetch poster URLs, keywords, and franchise data for ~25,000 titles. This requires a free API key.

**Steps:**
1. Create a free account at https://www.themoviedb.org/
2. Go to **Settings вҶ’ API** and generate an API key (v3 auth)
3. Create a `.env` file in the project root:

```bash
# .env
TMDB_API_KEY=your_api_key_here
```

> **Rate limit:** 40 requests per 10 seconds. Script 08 enforces this automatically. Expect it to run 30вҖ“60 minutes for the full catalog.

## Pipeline Details

### Script 01 вҖ” Clean Raw Data

**Reads:** 12 raw CSVs from `data/raw/`
**Writes:** 12 cleaned parquet files to `data/interim/`

- Parses `genres` and `production_countries` from string-encoded lists into Python lists
- Standardizes `type` field to `Movie` / `Show`
- Coerces numeric columns (`imdb_score`, `imdb_votes`, `runtime`, `release_year`)
- Adds `decade` bins (e.g. `2010s`) and `quality_tier` labels based on IMDb score
- Fills missing descriptions with empty string

---

### Script 02 вҖ” Merge Platforms

**Reads:** 12 interim parquets
**Writes:** 4 processed parquets to `data/processed/`

| Output File | Contents | Approx Rows |
|-------------|----------|-------------|
| `merged_titles.parquet` | Netflix + Max titles only | ~9K |
| `merged_credits.parquet` | Netflix + Max credits | ~27K |
| `all_platforms_titles.parquet` | All 6 platforms | ~25K |
| `all_platforms_credits.parquet` | All 6 platforms credits | ~60K |

---

### Script 03 вҖ” Compute Multi-Signal Similarity

**Reads:** `all_platforms_titles.parquet`
**Writes:** `data/precomputed/similarity/tfidf_top_k.parquet`, `models/tfidf_vectorizer.pkl`

Computes pairwise title similarity using 4 signals combined with learned weights:

| Signal | Weight | Method |
|--------|--------|--------|
| Description TF-IDF | 30% | Cosine similarity on description text (max 5,000 features, 1вҖ“2 ngrams) |
| Genre overlap | 30% | Cosine similarity on binary genre matrix |
| Same content type | 15% | Bonus if both Movie or both Show |
| IMDb score proximity | 25% | `1 - |scoreA - scoreB| / 10` |

Stores top 50 neighbors per title. Pages display the top 10 (configurable in `src/config.py`).

---

### Script 04 вҖ” Compute UMAP

**Reads:** `all_platforms_titles.parquet`, `models/tfidf_vectorizer.pkl`
**Writes:** `data/precomputed/dimensionality_reduction/umap_coords.parquet`, `models/umap_reducer.pkl`

- Combines TF-IDF description vectors + genre binary matrix (genre weighted 2Г—)
- UMAP params: `n_neighbors=15`, `min_dist=0.1`, `n_components=2`, `metric=cosine`
- Output used for the 2D catalog scatter plot in Explore Catalog

---

### Script 05 вҖ” Enrich with IMDb

**Reads:** `all_platforms_titles.parquet`, `data/raw/imdb/*.tsv.gz`
**Writes:** `data/enriched/imdb_enrichment.parquet`, `data/enriched/imdb_principals.parquet`

- Cross-references our catalog IMDb IDs against IMDb public files
- Extracts original title, adult content flag
- Builds per-title lists of directors, writers, producers, composers, cinematographers
- Streams IMDb files in 500K-row chunks to manage memory (~1 GB+ input)

---

### Script 06 вҖ” Enrich with Wikidata

**Reads:** `all_platforms_titles.parquet`, Wikidata SPARQL endpoint
**Writes:** `data/enriched/wikidata_enrichment.parquet`

- Batches IMDb IDs (50 per query) and sends SPARQL queries to `https://query.wikidata.org/sparql`
- Fetches: budget (USD), box office (USD), award wins, award nominations
- Caches each batch to `data/cache/wikidata/batch_XXXX.json` вҖ” **safe to interrupt and resume**
- Enforces 1-second rate limit per batch; auto-retries on HTTP 429

> **Expected coverage:** ~13% (awards); budget/box office even sparser (~10вҖ“20%). Low coverage is expected вҖ” not a bug.

---

### Script 07 вҖ” Enrich with MovieLens

**Reads:** `all_platforms_titles.parquet`, `data/raw/movielens/*.csv`
**Writes:** `data/enriched/movielens_genome.parquet`, `data/precomputed/embeddings/genome_vectors.npy`, `data/precomputed/embeddings/genome_id_map.parquet`

- Maps catalog movies to MovieLens via IMDb ID (`links.csv`)
- Extracts top 20 genome tags per movie from the tag relevance matrix
- Streams `ratings.csv` (509 MB) to compute per-movie average rating and count
- Stores dense genome vectors (movies Г— 1,128 tags) as a NumPy array for fast similarity lookups

> **Memory:** `genome-scores.csv` is 309 MB вҖ” ensure at least 4 GB free RAM.

---

### Script 08 вҖ” Enrich with TMDB

**Reads:** `all_platforms_titles.parquet`, TMDB API (requires `.env` with `TMDB_API_KEY`)
**Writes:** `data/enriched/tmdb_enrichment.parquet`

Makes 2 API calls per title:
1. `GET /find/{imdb_id}?external_source=imdb_id` вҖ” resolves IMDb ID вҶ’ TMDB ID
2. `GET /{movie|tv}/{tmdb_id}?append_to_response=keywords` вҖ” fetches keywords, franchise/collection, production companies, poster path

- Caches each response to `data/cache/tmdb/{imdb_id}.json` вҖ” **safe to interrupt and resume**
- Enforces 40 req/10s rate limit with sliding window; auto-retries on HTTP 429 after 10s

> **Expected run time:** 30вҖ“60 minutes for ~25K titles.

---

### Script 09 вҖ” Build Enriched Titles Master Table

**Reads:** `all_platforms_titles.parquet` + all 5 enrichment parquets
**Writes:** `data/enriched/titles_enriched.parquet`

Left-joins all enrichment sources on `imdb_id` to produce a single master table with all signals. Aggregates `imdb_principals` into per-title writer and producer name lists. Prints a coverage report on completion.

**Key columns added:**
```
original_title, is_adult,
budget_usd, box_office_usd, award_wins, award_noms,
top_tags, ml_avg_rating, ml_rating_count,
tmdb_keywords, collection_name, production_companies, poster_url,
imdb_writers, imdb_producers,
data_confidence
```

---

### Script 10 вҖ” Train Greenlight Predictor

**Reads:** `data/enriched/titles_enriched.parquet`
**Writes:** `models/greenlight_movie_predictor.pkl`, `models/greenlight_show_predictor.pkl`

Trains separate `GradientBoostingRegressor` models for movies and shows to predict IMDb score. Features include 19 genre binary columns, runtime, release year, country tier, franchise flag, and budget tier. Uses 5-fold cross-validation. Requires вүҘ200 samples per content type.

---

### Script 11 вҖ” Precompute Strategic Analysis

**Reads:** `data/enriched/titles_enriched.parquet`
**Writes:** `data/precomputed/strategic_analysis/prestige_index.parquet`, `data/precomputed/strategic_analysis/acquisition_targets.parquet`

- **Prestige index:** Award wins per 1,000 titles by platform Г— genre
- **Acquisition targets:** Genre gap analysis vs. each competitor with severity ratings, quality thresholds, and specific recommendations

---

### Script 12 вҖ” Precompute Cast & Crew Network

**Reads:** `all_platforms_credits.parquet`, `imdb_principals.parquet`, `all_platforms_titles.parquet`
**Writes:** `data/precomputed/network/edges.parquet`, `data/precomputed/network/person_stats.parquet`

- Merges platform credits with IMDb principals
- Filters to people with вүҘ3 titles; builds collaboration edges (co-appearances) with weight вүҘ2
- Computes NetworkX PageRank and Louvain community detection
- Stores per-person stats: title count, avg IMDb, career span, top genre, influence score, award context

## Troubleshooting

**Script 06 (Wikidata) stalls or returns HTTP 429**
Wikidata's SPARQL endpoint has burst limits. The script sleeps 1 second per batch and auto-retries. If it stalls, kill it and re-run вҖ” cached batches are skipped automatically.

**Script 07 (MovieLens) runs out of memory**
`genome-scores.csv` is 309 MB. Ensure вүҘ4 GB free RAM. The script streams `ratings.csv` (509 MB) so that file is fine.

**Script 08 (TMDB) is very slow**
Normal вҖ” 40 req/10s on ~25K titles takes 30вҖ“60 min. Cached responses are skipped on re-runs. If interrupted, just re-run the script; it picks up where it left off.

**Script 10 prints "skipping Movie/Show model вҖ” not enough samples"**
Fewer than 200 titles of that type had enrichment data. Check coverage output from Script 09.

**App page fails to load / KeyError**
At least one pipeline artifact is missing. Run `bash scripts/run_pipeline.sh` to completion before launching the app.

**Poster images don't appear**
`poster_url` comes from TMDB (Script 08). If Script 08 was skipped or incomplete, ~95% of posters will be missing. Re-run Script 08 and then Script 09 to rebuild the master table.
